# Sprite System Guide

## Overview

This project uses a sprite manifest system that supports both external image files and AI-generated sprites. The system is designed to make it easy to add sprites generated by AI tools like nano-banana, DALL-E, Midjourney, etc.

## Sprite Categories

- **NPC**: Non-player characters (guides, traders, shamans)
- **ITEM**: Collectible items (harpoons, medicine, maps)
- **CREATURE**: Animals and wildlife (piranhas, jaguars, macaws)
- **ICON**: UI icons (health, hunger, location markers)
- **PORTRAIT**: Character portraits (larger, detailed images)
- **ENVIRONMENT**: Background elements (trees, rocks, plants)
- **UI**: UI elements and decorations

## Adding AI-Generated Sprites

### Option 1: Base64 Data URL (Embedded)

Perfect for small sprites or when you don't want external files:

```typescript
{
  id: 'npc_guide_new',
  category: SpriteCategory.NPC,
  name: 'New Guide',
  src: 'data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...', // Full base64 string
  width: 64,
  height: 64,
  generatedBy: 'nano-banana',
  prompt: 'Elderly Amazon river guide with gray hair',
  generatedAt: '2026-01-23T10:00:00Z',
}
```

### Option 2: External File

Better for larger sprites or when you want to manage files separately:

```typescript
{
  id: 'creature_capybara',
  category: SpriteCategory.CREATURE,
  name: 'Capybara',
  src: '/sprites/creatures/capybara.png', // Place file in public/sprites/creatures/
  width: 48,
  height: 48,
  generatedBy: 'midjourney',
  prompt: 'Friendly capybara swimming in Amazon river, pixel art style',
}
```

## Using AI Tools

### With nano-banana (or similar CLI tools):

```bash
# Generate sprite
nano-banana generate "elderly Amazon river guide" --size 64x64 --style pixel-art

# Convert to base64
base64 output.png

# Or save to public directory
cp output.png public/sprites/npc/guide_elderly.png
```

### With DALL-E or Midjourney:

1. Generate image with your prompt
2. Download the image
3. Resize to appropriate dimensions (32x32, 48x48, 64x64, 128x128)
4. Either:
   - Convert to base64 and embed in manifest
   - Save to `public/sprites/{category}/` directory

## Recommended Sprite Sizes

- **Icons**: 24x24 or 32x32
- **Items**: 32x32 or 48x48
- **Creatures**: 48x48 or 64x64
- **NPCs**: 64x64
- **Portraits**: 128x128 or 256x256
- **Environment**: Variable (48x48 to 128x128)

## Art Style Guidelines

For consistency, consider:

- **Pixel Art**: 16-bit or 32-bit style (SNES/GBA era)
- **Color Palette**: Earthy tones (greens, browns, blues, golds)
- **Perspective**: Top-down or 3/4 view for consistency
- **Background**: Transparent PNG for compositing

## Testing Your Sprites

After adding sprites to the manifest, test them:

```typescript
// In your component or game code
const spriteManager = new SpriteManager()
spriteManager.setManifest(spriteManifest)

// Load and display
const sprite = await spriteManager.loadSprite('npc_guide_elderly')
if (sprite.loaded) {
  // Use sprite.texture with PixiJS
}
```

## Batch Generation Script (Optional)

Create a script to generate all sprites at once:

```bash
#!/bin/bash
# generate-sprites.sh

# NPCs
nano-banana generate "elderly Amazon guide" --output sprites/npc/guide_elderly.png
nano-banana generate "river merchant trader" --output sprites/npc/trader_merchant.png

# Creatures
nano-banana generate "piranha fish" --output sprites/creatures/piranha.png
nano-banana generate "jaguar predator" --output sprites/creatures/jaguar.png

# Items
nano-banana generate "brass compass icon" --output sprites/items/compass.png
nano-banana generate "harpoon tool" --output sprites/items/harpoon.png
```
